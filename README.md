# StockExperimentAI Live Picks — Tamper-Evident Public Record

This repository is a public record of the picks generated by an ongoing machine-learning portfolio experiment. Every rebalance, the model selections are committed here and cryptographically timestamped via [OpenTimestamps](https://opentimestamps.org/) to provide a falsifiable audit trail.

The point: prove that the picks weren't backfilled with the benefit of hindsight.

## What this repo proves (and doesn't)

**This repo PROVES:**
- Each `picks/MMDDYYYY.csv` file was committed publicly on (or shortly after) the date claimed
- The SHA-256 hash of each picks file existed before the corresponding Bitcoin block confirmation referenced by its `.ots` proof
- The file's contents cannot have been altered after the OpenTimestamps anchor

**This repo does NOT prove:**
- That the underlying model wasn't retrained or tweaked AFTER seeing the picks
  (you'd have to trust the operator on that, or audit the source repo)
- The 2026-03-02 picks file is **not cryptographically anchored** — it was committed
  retroactively from local DB state. Only files from 2026-04-30 onward have
  OpenTimestamps proofs (`.ots` siblings).

## The four portfolios

Each portfolio starts with $1M of (theoretical) capital. Through 2026-04-30 the experiment runs four portfolios; on that date, `ml-model-1` is retired and replaced with `ensemble-cb`.

| Slug | Model | Description |
|---|---|---|
| `ensemble` | XGB80/GE20 | Live incumbent ensemble — XGBoost 80% + GrowthEvaluator 20% |
| `ensemble-cb` | XGB60/GE20/CB20 | NEW as of 2026-04-30 — adds CatBoost to the production blend |
| `growth-evaluator` | GrowthEvaluator v3.2 | Rule-based: growth 50%, profitability 40%, earnings quality 10% |
| `ml-model-2` | LightGBM v1 Trial 33 | Standalone gradient boosting |
| `ml-model-1` (retired 2026-04-30) | XGBoost v5 Trial 33 | Standalone XGB — archived because of high overlap with `ensemble` |

## Rebalance cadence

Every 42 trading days (~9 weeks). Confirmed dates:
- 2026-03-02 — initial fill (Mon)
- 2026-04-30 — first rebalance (Thu)
- 2026-07-01 — next (Wed)
- 2026-08-31 — (Mon)
- 2026-10-29 — (Thu)
- 2026-12-30 — (Wed)

Picks are generated the day before each fill (after market close), recorded here on the morning of the fill, and timestamped within hours.

## File schema

Each `picks/MMDDYYYY.csv` contains 60 rows = 4 portfolios × 15 holdings.

| Column | Meaning |
|---|---|
| `ticker` | S&P 500 ticker |
| `portfolio` | slug — one of `ensemble`, `ensemble-cb`, `growth-evaluator`, `ml-model-2` |
| `fill_date` | YYYY-MM-DD |
| `shares` | actual shares purchased (sized to portfolio capital + equal-weight allocation) |
| `entry_price` | open price on `fill_date` |
| `sector` | GIC sector |
| `xgb_score`, `xgb_rank` | XGBoost percentile + cross-sectional rank |
| `ge_score`, `ge_rank` | GrowthEvaluator |
| `cb_score`, `cb_rank` | CatBoost |
| `lgb_score`, `lgb_rank` | LightGBM |
| `ensemble_score`, `ensemble_rank` | XGB80/GE20 blended |
| `ensemble_cb_score`, `ensemble_cb_rank` | XGB60/GE20/CB20 blended |

## Verification

Each picks CSV from 2026-04-30 onward has a corresponding `picks/MMDDYYYY.csv.ots` OpenTimestamps proof.

To verify a file:

```bash
# 1. Install the OpenTimestamps client (one-time)
pip install opentimestamps-client

# 2. Verify (e.g.)
ots verify picks/04302026.csv.ots
# Output:
#   Got 1 attestation(s) from blockchain.info
#   Bitcoin block 920000 attests data existed as of 2026-04-30 13:24 UTC
```

The OpenTimestamps proof anchors the SHA-256 hash of the CSV to a Bitcoin block header. Anyone can verify that the CSV's exact bytes existed before that block was mined — making backdating impossible without compromising Bitcoin itself.

For the 2026-03-02 file (no `.ots`), the only available evidence is the Git commit history of this repo and the Git commit history of the [main project repo](https://github.com/doubledown3301/StockExperimentAI). Treat its provenance accordingly.

## How picks are generated (pipeline summary)

1. **Wednesday evening** before each rebalance: production database snapshot exported (prices, fundamentals, earnings).
2. **Wednesday evening:** Models trained on data through scoring date. Top-15 picks per portfolio. (See [picks_generation.ipynb](https://github.com/doubledown3301/StockExperimentAI/blob/master/BackTests/Live_Scoring/picks_generation.ipynb) in the main project repo.)
3. **Thursday morning at market open:** Picks executed via `populate_real_portfolio.py`. Archive CSV written to local `BackTests/Live_Scoring/archive/MMDDYYYY.csv`.
4. **Thursday morning, post-execution:** This public repo updated via `scripts/sync_picks_public.py`. Push triggers GitHub Actions, which runs `ots stamp` and commits the `.ots` proof back.
